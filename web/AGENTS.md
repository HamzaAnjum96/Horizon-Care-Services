# Brand asset generation

Brand PNGs in `web/public/brand/` are generated by `scripts/generate-brand-pngs.cjs`.
Run it from the repo root: `node scripts/generate-brand-pngs.cjs`

## Prerequisites
`web/.next/static/media/` must exist and contain the Source Serif 4 woff2 file.
If it doesn't, run `npm run build` inside `web/` first.

## Why Playwright instead of sharp/librsvg
Text assets (logos, merch, banners) are rendered via Playwright/Chromium, **not** sharp.
Reason: librsvg ignores CSS `font-variation-settings`, so variable font axes (`opsz`, `wght`)
have no effect — exported text looks wrong (wrong weight, wrong tracking).
Chromium renders the axes correctly.

## Why localhost font server
Chromium silently drops `@font-face` rules that use base64 `data:` URIs for woff2 files.
The script starts a temporary `http.createServer` on a random port and serves the woff2
from there. Font loading is confirmed with `document.fonts.check()` before screenshotting.

## Font settings (must match `branding-grid.tsx` exactly)
| Asset            | font-variation-settings         | Notes                          |
|------------------|---------------------------------|--------------------------------|
| Stacked name     | `opsz 14, wght 560`             | letter-spacing -0.01em         |
| Lockup name      | `opsz 16, wght 560`             | letter-spacing -0.01em         |
| Pen name         | `opsz 14, wght 600`             |                                |
| Pen URL          | `opsz 10, wght 600`             | same colour as logo, no opacity |
| Pen phone        | `opsz 10, wght 600`             | same colour as logo, no opacity |
| Mug name         | `opsz 14, wght 600`             |                                |
| Mug URL/phone    | `opsz 10, wght 600`             | same colour as logo, no opacity |
| Banner headline  | `opsz 64, wght 620`             |                                |
| Banner logo name | `opsz 64, wght 580`             |                                |

## Layout scale factor
Preview card height is 68 px. Canvas height is 300 px (pen) or varies per asset.
Scale factor `S = canvas_height / 68` converts all preview pixel values to canvas size,
keeping proportions identical between the live preview and exported PNG.

## Playwright / Chromium paths
- Chromium binary: `/opt/pw-browsers/chromium-1194/chrome-linux/chrome`
- Playwright module: `/opt/node22/lib/node_modules/playwright`

## Output resolutions
- 1× HD files (`*.png`, `*-tr.png`): `deviceScaleFactor: 1`
- 2× files (`*@2x.png`, `*-tr@2x.png`): `deviceScaleFactor: 2`
- 4× Ultra HD files (`*@4x.png`): `deviceScaleFactor: 4`
